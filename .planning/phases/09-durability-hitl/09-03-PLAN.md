## Task 1: Add idempotency key to job creation

**Files:** `src/api/jobs.py`, `src/schemas/jobs.py`

### Context

If a user submits the same job twice (e.g., re-clicking a button, network retry), the system should detect this and return the existing job result instead of creating a duplicate.

### Action

Add `idempotency_key` to `Job` model and job creation API:

1. Add `idempotency_key` column to `Job` model:
```python
idempotency_key: Mapped[str | None] = mapped_column(String(255), nullable=True, unique=True)
```

2. Add to job creation schema:
```python
class CreateJobRequest(BaseModel):
    job_type: str
    idempotency_key: str | None = None  # Client-provided dedup key
    priority: int | None = None
    input_data: dict[str, Any] | None = None
    workspace_id: UUID | None = None
```

3. In `POST /jobs` endpoint:
```python
if request.idempotency_key:
    # Check for existing job with same key and same job_type
    existing = await db.execute(
        select(Job).where(
            Job.idempotency_key == request.idempotency_key,
            Job.job_type == request.job_type,
        )
    )
    existing_job = existing.scalar_one_or_none()
    if existing_job:
        # Return existing job instead of creating duplicate
        return {
            "id": str(existing_job.id),
            "status": existing_job.status,
            "result_data": existing_job.result_data,
            "idempotent": True,  # Tell client this was a dedup
        }
```

Create migration:
```python
"""Add idempotency_key to jobs table."""
op.add_column("jobs", sa.Column("idempotency_key", sa.String(255), nullable=True, unique=True))
```

### Acceptance Criteria
- [ ] Duplicate submission with same idempotency_key returns existing job
- [ ] `idempotent: True` in response indicates deduplication
- [ ] Different job types with same key create separate jobs
